    *2 How would you describe the difference between HTML, CSS, and Javascript?

    HTML to create the actual content of the page, this means in HTML we can define the basic structure and the contents of a website. HTML allows you to put images, text, videos, forms and other pieces of content together into the webpage.

    A CSS file contains the styling of the page. It allows you to change colors, positioning and more. It is kind of like the design of the building itself. CSS is used to control presentation, formatting, and layout.

    JavaScript is programming language designed to make web development easier and more attractive. JavaScript is the text-based programming language used both on the client-side and server-side that allows you to create responsive, interactive elements for web pages, enhancing the user experience

    *3 How would you briefly describe frontend to someone who has no experience?

    Front-end web development, also known as client-side development is the practice of producing HTML, CSS, and JavaScript for a website or Web Application so that a user can see and interact with them directly.

    The objective of front-end design is to ensure that when the users open up the site they see the information in a format that is easy to read and relevant. Users nowadays use a large variety of devices with varying screen sizes and resolutions thus forcing the designer to take into consideration these aspects when designing the site. They need to ensure that their site comes up correctly in different browsers (cross-browser), different operating systems (cross-platform) and different devices (cross-device).

    Front-end developer should closely work with UX designer to meet good user experience in front-end.

    *4 Why is semantically correct HTML important?

    Semantic HTML is really important for SEO, Accessibility and cross-browser compatibility.

    To implement an accessible website, semantic structural is most important. semantic HTML is the key starting point toward good accessibility practices, mainly when a screen reader or any sort of assistive device scans a web page, it gets information about the Document Object Model (DOM), or the HTML structure of the page. No styles or JavaScript will be read by a screen reader. So its really important to develop a structure with semantic HTML tags.

    Browsers interact with semantic elements with their own style and behaviors. "Valid HTML" is important for compatibility. It points to accessibility and web standards. The site will emphasize you about the semantic markup to ensure cross browser accessibility.

    Importance of semantic HTML evolved dramatically in SEO to fulfill Google's mission. Along with all SEO tips and tricks, content should be equipped with correct tags maintaining a clear hierarchy and architecture. This made it impossible for search engines to identify the correct web page content

    Below is an example of Semantic basic HTML structure

      
<header> 
	<nav>Header Nav links</nav>
</header>
<main>
	<h1>Main Heading</h1>
	<aside>Left </aside>
	<section>Right</section>
        <article>Bottom</article>
</main>

<footer>
<nav>Footer Nav links</nav>
</footer>
